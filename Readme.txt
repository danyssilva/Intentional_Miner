# Intentional Miner and Solutions Quality Evaluation – Repository Structure

## Overview

This repository contains two main components:

1. **Intentional Miner Implementation** – responsible for extracting SBMN constraints and generating models.
2. **Solutions Quality Evaluation** – used to evaluate the quality of generated BPMN models based on event logs.

All datasets and files have been anonymized for research purposes.

---

## 1. Intentional Miner Repository

This directory contains the implementation of the **Intentional Miner**, along with example outputs generated during the experiments.

### Root Files

* `processing_constraints.py` – Functions for extracting and handling SBMN constraints.
* `sbmn_model_functions.py` – Core functions for building and manipulating SBMN models.
* `templates_groups.py` – Definition and grouping of constraint templates.
* `validating_functions.py` – Validation routines for generated models.
* `printing_functions.py` – Utilities for formatting and displaying results.
* `ReadSBMN.py` – Functions for reading SBMN models from files.

---

### OUTPUT/

This folder stores the results generated by the Intentional Miner for different case studies.

#### Structure:

```
OUTPUT/
 ├── <CaseStudyName>/
      ├── alpha_miner/
      │     └── alpha_miner_RM_<Case>.bpmn
      ├── heuristic_miner/
      │     └── heuristic_miner_RM_<Case>.bpmn
      ├── inductive_miner/
      │     └── inductive_miner_RM_<Case>.bpmn
      ├── sbmn_RM_<Case>.json
      └── sbmn_RM_<Case>.txt
```

#### Description:

* **alpha_miner / heuristic_miner / inductive_miner**

  * BPMN models discovered using traditional process mining algorithms.

* **sbmn_RM_<Case>.json**

  * Structured SBMN model extracted by the Intentional Miner.

* **sbmn_RM_<Case>.txt**

  * Readable representation of the extracted constraints.

#### Example of available Case Studies:

* `ThesisDefense`
* `Vacation`

## 2. Solutions Quality Evaluation Repository

This directory contains the implementation used to evaluate the quality of generated BPMN models using event logs.

---

### Root Files

* `evaluate_bpmn_models_pm4py.py`

  * Evaluation using PM4Py (precision, fitness, generalization, simplicity) directly from BPMN reference models.

* `evaluate_bpmn_models_xes_multithread.py`

  * Evaluation using PM4Py (precision, fitness, generalization, simplicity) directly from .xes archives. Multithreaded evaluation for improved performance.

* `eventlog_generator.py`

  * Utility for generating event logs from models.

---

### INPUT/

This folder contains the datasets used in the evaluation experiments.

#### Structure:

```
INPUT/
 └── PAPER/
      ├── <DatasetName>/
```

---

### Dataset Structure

Each dataset (e.g., `CaseHandling`, `Complaint`) follows this structure:

```
<DatasetName>/
 ├── alpha_miner/
 │     └── alpha_miner_RM_<dataset>.bpmn
 ├── heuristic_miner/
 │     └── heuristic_miner_RM_<dataset>.bpmn
 ├── inductive_miner/
 │     └── inductive_miner_RM_<dataset>.bpmn
 ├── RM_<dataset>.xes
 ├── proc_sbmn_IM_<dataset>.log
 ├── S*_proc_sbmn_IM_<dataset>.bpmn
 └── S*_proc_sbmn_IM_<dataset>.json
```

---

### Description of Files

* **alpha_miner / heuristic_miner / inductive_miner**

  * Reference BPMN models discovered using classical mining algorithms.

* **RM_<dataset>.xes**

  * Original event log used as ground truth.

* **proc_sbmn_IM_<dataset>.log**

  * Log generated or processed by the Intentional Miner.

* **S**proc_sbmn_IM*<dataset>.bpmn**

  * BPMN models generated from SBMN constraints (candidate solutions).

* **S**proc_sbmn_IM*<dataset>.json**

  * Corresponding SBMN representations of each generated solution.

---

### Naming Convention for Solutions

* `S<number>` represents a specific generated solution.
* Each solution includes:

  * A BPMN model (`.bpmn`)
  * Its corresponding SBMN representation (`.json`)

---

## General Workflow

1. Event logs (`.xes`) are used as input.
2. The Intentional Miner extracts SBMN constraints.
3. Multiple BPMN models are generated based on these constraints by any transformation algorithm.
4. The generated models are evaluated against the reference log.
5. Quality metrics are computed for comparison.

---

## Notes

* All datasets were anonymized for research purposes.
* File naming conventions were standardized to ensure reproducibility.
* The structure supports automated evaluation pipelines.
